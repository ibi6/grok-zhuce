# Turnstile 自动跳过设计

## 目标

当注册或 CPA OAuth 流程检测到需要人工完成的 Turnstile/Cloudflare 人机验证时，不再提示或等待用户手动处理，而是自动终止当前账号尝试、记录原因并继续后续任务。

## 行为

1. 页面仅加载 Turnstile 脚本但没有可见挑战时，继续正常流程。
2. 检测到可见挑战、验证超时或 Cloudflare 挑战中间页时，抛出统一的“需要人机验证”结果。
3. 当前账号立即停止，不模拟点击、不 reset challenge、不调用打码平台。
4. 释放 Outlook 邮箱租约并关闭当前浏览器会话。
5. 该次计入失败，日志显示“检测到人机验证，已自动跳过”。
6. 单任务连续出现 3 次人机验证后冷却 60 秒，避免快速重复触发。
7. CPA OAuth 阶段遇到挑战时只判定 CPA 导出失败；已经成功注册的账号和 SSO 仍正常保存。

## 不包含

- 自动破解、模拟通过或绕过 Turnstile。
- YesCaptcha 等第三方打码服务。
- 为规避风控而切换代理或提高并发。
- 删除现有人工可见浏览器模式；用户仍可关闭自动跳过配置后自行处理。

## 配置

新增配置：

```json
{
  "turnstile_auto_skip": true,
  "turnstile_skip_cooldown_after": 3,
  "turnstile_skip_cooldown_seconds": 60
}
```

默认启用自动跳过。GUI 注册页显示“遇到人机验证自动跳过”开关。

## 测试与验收

1. 仅加载脚本不会被误判为挑战。
2. 可见挑战会产生统一自动跳过异常。
3. GUI 和 CLI 都不会进入人工等待。
4. 连续挑战达到阈值后执行可取消冷却。
5. CPA 挑战不会阻止已注册账号保存。
6. 全部测试、PyInstaller 构建和 EXE 启动检查通过。
